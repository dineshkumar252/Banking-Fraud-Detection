Project Overview
The Banking Fraud Detection & Customer Analytics project is a SQL-based data analytics project designed to simulate real-world banking operations and demonstrate practical database management and analytical skills. The project focuses on detecting fraudulent transactions, analyzing customer behavior, and generating meaningful business insights using structured banking data.
The project consists of two relational tables: Customers and Transactions, connected through Primary Key and Foreign Key relationships. Using SQL, customer and transaction data is analyzed to identify fraud, monitor transaction patterns, categorize customers based on account balances, and generate analytical reports for decision-making.
This project demonstrates a wide range of SQL concepts, including CREATE DATABASE, CREATE TABLE, INSERT, SELECT, WHERE, ORDER BY, GROUP BY, HAVING, Aggregate Functions, CASE Statements, INNER JOIN, LEFT JOIN, RIGHT JOIN, Subqueries, EXISTS, ANY, ALL, Views, Common Table Expressions (CTEs), Window Functions (ROW_NUMBER, RANK, DENSE_RANK, NTILE, LEAD, LAG), Stored Procedures, Triggers, and Indexes.
Key analyses performed include identifying fraudulent transactions, calculating total fraud amounts, ranking high-value transactions, customer segmentation (Regular, Premium, and VIP), city-wise customer distribution, average account balance analysis, and generating business reports for fraud monitoring and customer analytics.
This project strengthened my understanding of relational database design, advanced SQL querying, data analysis, fraud detection techniques, and business reporting. It demonstrates how SQL can transform raw banking data into actionable insights, making it a valuable portfolio project for Data Analyst, Business Analyst, and SQL Developer roles.
Tools & Technologies: MySQL, SQL, MySQL Workbench, Git, GitHub.
